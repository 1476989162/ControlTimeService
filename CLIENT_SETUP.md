# 客户端连接控制端配置指南

## 问题说明

如果您已经启动了控制端（服务端），但打开客户端后控制端没有显示客户端列表，这是因为**客户端缺少配置文件**，不知道要连接到哪个服务器。

## 解决步骤

### 第一步：获取控制端的IP地址和端口

1. **在控制端电脑上查看IP地址：**
   - 按 `Win + R`，输入 `cmd`，回车
   - 输入命令：`ipconfig`
   - 找到 "IPv4 地址"，例如：`192.168.1.100`

2. **确认控制端监听的端口：**
   - 启动控制端后，窗口底部会显示："服务器已启动，监听端口 XXXX"
   - 默认端口是 **9528**（如果9528被占用，可能会自动切换到其他端口）

### 第二步：创建客户端配置文件

在**客户端电脑**上，找到客户端程序所在目录，创建 `control_config.json` 文件：

**文件位置：**
```
ControlTimeService\bin\Debug\net8.0-windows\control_config.json
```

**文件内容：**
```json
{
  "server_url": "http://控制端IP地址:端口号"
}
```

**示例：**
如果控制端IP是 `192.168.1.100`，端口是 `9528`，则内容为：
```json
{
  "server_url": "http://192.168.1.100:9528"
}
```

### 第三步：重启客户端

1. 关闭正在运行的客户端程序
2. 重新启动客户端
3. 等待几秒钟，客户端会自动注册到控制端

### 第四步：验证连接

在控制端界面上：
1. 点击 **"刷新客户端列表"** 按钮
2. 应该能看到客户端出现在列表中
3. 显示客户端名称、IP地址、状态等信息

## 常见问题排查

### 问题1：配置文件创建后仍然无法连接

**检查清单：**
- [ ] 确认文件名是 `control_config.json`（不是 `.txt`）
- [ ] 确认文件编码是 UTF-8
- [ ] 确认JSON格式正确（可以使用在线JSON验证工具）
- [ ] 确认IP地址和端口号正确

### 问题2：客户端和控制端不在同一台电脑

**网络要求：**
- 两台电脑必须在**同一个局域网**内
- 可以互相ping通

**测试方法：**
在客户端电脑上打开CMD，执行：
```bash
ping 控制端IP地址
```

如果能收到回复，说明网络连通。

### 问题3：防火墙阻止连接

**解决方法：**
1. 在控制端电脑上打开 "Windows Defender 防火墙"
2. 点击 "高级设置"
3. 点击 "入站规则" → "新建规则"
4. 选择 "端口" → 下一步
5. 选择 "TCP"，特定本地端口输入：`9528`（或实际使用的端口）
6. 选择 "允许连接"
7. 完成

或者临时关闭防火墙测试（不推荐长期使用）。

### 问题4：控制端端口不是9528

如果控制端启动时显示其他端口（如9529、8080等），需要在配置文件中修改：

```json
{
  "server_url": "http://192.168.1.100:9529"
}
```

### 问题5：客户端日志调试

如果需要查看客户端的连接日志：

1. 在客户端程序目录下找到 `ControlTimeService.exe`
2. 右键 → 属性 → 兼容性
3. 暂时取消 "以管理员身份运行"（仅用于调试）
4. 从命令行启动：
   ```bash
   cd ControlTimeService\bin\Debug\net8.0-windows
   .\ControlTimeService.exe
   ```
5. 查看控制台输出或调试信息

## 快速配置脚本

您可以使用以下PowerShell脚本快速创建配置文件：

```powershell
# 在客户端电脑上运行此脚本
$serverIP = Read-Host "请输入控制端IP地址"
$serverPort = Read-Host "请输入控制端端口（默认9528）"

if ([string]::IsNullOrWhiteSpace($serverPort)) {
    $serverPort = "9528"
}

$config = @{
    server_url = "http://${serverIP}:${serverPort}"
}

$config | ConvertTo-Json | Out-File -FilePath ".\control_config.json" -Encoding UTF8

Write-Host "配置文件已创建！" -ForegroundColor Green
Write-Host "服务器地址: http://${serverIP}:${serverPort}" -ForegroundColor Cyan
```

**使用方法：**
1. 复制上述代码保存为 `setup_client.ps1`
2. 在客户端程序目录中右键 → "使用PowerShell运行"
3. 按提示输入控制端IP和端口

## 配置示例

### 示例1：控制端和客户端在同一台电脑（测试用）
```json
{
  "server_url": "http://localhost:9528"
}
```

### 示例2：控制端IP为192.168.1.100
```json
{
  "server_url": "http://192.168.1.100:9528"
}
```

### 示例3：控制端使用8080端口
```json
{
  "server_url": "http://192.168.1.100:8080"
}
```

## 连接成功后

连接成功后，您可以在控制端进行以下操作：

1. **查看客户端状态**
   - 在线/离线状态
   - 剩余使用时间
   - 是否处于锁定状态

2. **远程控制**
   - 远程锁定客户端
   - 远程解锁客户端
   - 远程关闭客户端

3. **编辑配置**
   - 右键客户端 → "编辑配置"
   - 设置每天的使用时间规则
   - 点击 "保存并应用" 推送到客户端

## 需要帮助？

如果按照以上步骤仍然无法连接，请提供以下信息：

1. 控制端IP地址和端口号
2. 客户端 `control_config.json` 文件内容
3. 两台电脑是否能互相ping通
4. 控制端和客户端的Windows版本
5. 是否有安装第三方杀毒软件或防火墙
