# 控制端启动问题排查指南

## 常见问题：服务端打开时提示"拒绝打开"

### 问题原因分析

控制端启动失败通常由以下三个原因导致：

1. **权限不足** - HTTP服务器监听端口需要管理员权限
2. **端口被占用** - 默认端口（9528）可能被其他程序占用
3. **防火墙阻止** - Windows防火墙可能阻止了端口监听

### 解决方案

#### 方案一：以管理员身份运行（推荐）

这是最常见且最有效的解决方法：

1. **右键点击** `ControlCenter.exe`
2. 选择 **"以管理员身份运行"**
3. 如果弹出UAC提示，点击 **"是"**

**如何设置始终以管理员身份运行：**
1. 右键点击 `ControlCenter.exe` → 属性
2. 切换到 **"兼容性"** 选项卡
3. 勾选 **"以管理员身份运行此程序"**
4. 点击 **"应用"** → **"确定"**

#### 方案二：检查并关闭占用端口的程序

如果端口被占用，系统会自动尝试其他端口（9529, 9530, 8080, 8081等）。

**查看端口占用情况：**
```powershell
# 打开PowerShell或CMD，执行：
netstat -ano | findstr "9528"
netstat -ano | findstr "8080"
```

如果发现端口被占用，可以：
- 关闭占用端口的程序
- 或者让系统自动切换到其他可用端口（已实现）

#### 方案三：配置Windows防火墙

如果防火墙阻止了端口监听：

1. 打开 **"Windows Defender 防火墙"**
2. 点击 **"高级设置"**
3. 点击 **"入站规则"** → **"新建规则"**
4. 选择 **"端口"** → 下一步
5. 选择 **"TCP"**，特定本地端口输入：`9528,8080,8081,9529`
6. 选择 **"允许连接"**
7. 选择适用的网络类型（域、专用、公用）
8. 命名规则："时间控制系统-控制端"
9. 完成

### 改进功能说明

为了解决这个问题，我们已经做了以下改进：

#### 1. 添加管理员权限请求
- 创建了 `app.manifest` 文件
- 设置了 `<requestedExecutionLevel level="requireAdministrator" />`
- 程序启动时会请求管理员权限

#### 2. 自动端口切换
- 如果默认端口（9528）被占用，会自动尝试其他端口
- 尝试顺序：9528 → 9529 → 9530 → 8080 → 8081
- 成功后会显示实际使用的端口号

#### 3. 详细的错误提示
现在启动失败时会显示具体的错误原因和解决方法：

**权限不足时：**
```
权限不足：无法启动 HTTP 服务器。请以管理员身份运行此程序...

解决方法：
1. 右键点击程序 -> '以管理员身份运行'
2. 或者在防火墙设置中允许此程序
3. 关闭占用端口的其他程序
```

**端口不可用时：**
```
无法启动 HTTP 服务器。所有尝试的端口 (9528, 9529, 9530, 8080, 8081) 都不可用。

可能原因：
1. 端口被其他程序占用
2. 没有管理员权限
3. 防火墙阻止了端口监听

建议：以管理员身份运行此程序，或关闭占用端口的其他程序。
```

   - 启动客户端，应该能成功连接

### 常见问题FAQ

**Q: 为什么需要管理员权限？**
A: HTTP服务器监听网络端口（特别是使用 `http://*:port/` 绑定所有网络接口）需要管理员权限。这是Windows的安全机制。

**Q: 能否不使用管理员权限运行？**
A: 可以修改为只监听localhost（`http://localhost:port/`），但这样局域网内的其他电脑就无法连接了。对于控制端-客户端架构，必须监听所有网络接口。

**Q: 端口被占用了怎么办？**
A: 系统已经实现了自动端口切换功能，会尝试多个端口直到找到可用的。您也可以在代码中修改默认端口号。

**Q: 如何查看当前监听的端口？**
A: 启动成功后，窗口底部状态栏会显示 "服务器已启动，监听端口 XXXX"。

**Q: 客户端连接不上控制端怎么办？**
A: 
1. 确认控制端已成功启动（状态显示"运行中"）
2. 确认客户端的 `control_config.json` 中IP地址和端口正确
3. 确认两台电脑在同一局域网内
4. 检查防火墙是否阻止了通信
5. 在客户端电脑上ping控制端IP，确认网络连通
